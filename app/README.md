# App 目录分层说明

> 状态：迁移骨架已建立
> 更新：2026-04-18

当前仓库处于从“平铺式 `app/core`”向“分层 + 分域”结构迁移的过渡阶段。

## 1. 目标顶层结构

- `presentation/`：API、UI、对外输入输出适配
- `application/`：用例、编排、流程、任务路由
- `domain/`：领域对象、业务规则、策略接口
- `infrastructure/`：LLM、检索、存储、执行、安全、观测
- `shared/`：跨层共享 DTO / types / utils
- `config/`：环境和配置

## 2. 迁移规则

1. 新能力优先进入新分层目录，不再继续平铺到 `app/core/`
2. 旧代码允许暂时保留，但逐专题迁移
3. `presentation` 不直接承载复杂业务逻辑
4. `domain` 不依赖具体 provider、数据库连接器或框架实现
5. `infrastructure` 只实现技术细节，不决定业务流程

## 3. 过渡期说明

当前以下目录仍然存在并会逐步迁移：

- `api/`
- `ui/`
- `core/`

目标不是立刻删除它们，而是在后续迭代中把新增和重构后的代码逐步迁到新分层目录中。
