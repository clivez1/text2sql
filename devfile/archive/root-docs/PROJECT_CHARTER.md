# Dify Text2SQL Agent 项目章程

## 1. 项目目标

### 核心目标
构建一个**可展示的技术作品**，证明在 BI+AI 领域的开发能力，用于求职展示。

### 具体目标
- 实现自然语言到 SQL 的准确转换
- 支持主流数据库（SQLite/PostgreSQL/MySQL）
- 提供直观的数据可视化结果
- 符合企业级安全标准
- 代码质量达到生产级水平

## 2. 项目需求

### 功能需求
| 优先级 | 需求 | 说明 |
|--------|------|------|
| P0 | 自然语言转SQL | 核心功能，支持常见查询场景 |
| P0 | 多数据库支持 | SQLite（Demo）、PostgreSQL、MySQL |
| P1 | 数据可视化 | 自动生成图表（柱状图、折线图、饼图） |
| P1 | 安全校验 | SQL注入防护、只读权限、敏感字段过滤 |
| P2 | 报告导出 | PDF/Excel格式导出查询结果 |
| P2 | 通知集成 | 企业微信/钉钉结果推送 |

### 非功能需求
| 类别 | 要求 | 说明 |
|------|------|------|
| 技术栈 | Python为主 | 无算法门槛，纯开发导向 |
| 性能 | 响应时间<3s | 包含LLM调用时间 |
| 安全性 | 企业级标准 | 防注入、权限控制 |
| 可维护性 | 模块化设计 | 易于扩展和修改 |
| 文档 | 完整文档 | README + API文档 + 使用示例 |

## 3. 项目边界

### 包含范围 ✅
- **技术作品开发**：完整的Text2SQL Agent实现
- **求职展示材料**：可运行的Demo、技术文档、架构说明
- **核心功能**：NL2SQL、多数据库、可视化、安全校验
- **部署方案**：Docker容器化、云服务部署指南

### 排除范围 ❌
- **简历生成功能**：不涉及任何AI简历相关功能
- **复杂算法开发**：不涉及机器学习模型训练
- **大而全的平台**：聚焦核心功能，不做通用平台
- **生产环境运维**：提供部署方案但不负责实际运维
- **商业产品化**：仅作为技术作品，不追求商业化

## 5. 约束条件

### 技术约束
- 必须使用 Python 开发
- LLM 适配层必须支持 Bailian Code Plan 和 OpenAI Compatible
- 数据库操作必须通过安全抽象层

### 资源约束
- 开发周期：4周（Week1-Week4）
- 依赖现有 Dify 生态工具
- 不需要额外硬件资源

### 风险约束
- LLM API 成本控制
- Dify 平台版本兼容性
- 开源项目竞争差异化
