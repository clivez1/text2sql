# 16. 数据可视化与分析表达层方案

> 状态：当前生效
> 更新：2026-04-18

---

## 1. 目标

把当前“查询结果 + 自动推荐图表”升级为“分析结果表达层”。

目标不是做完整 BI 设计器，而是让 Copilot 的输出从数据表格升级为：

- 数据答案
- 解释
- 图表计划
- 图表 spec
- 可导出工件

---

## 2. 当前差距

当前图表层主要基于列类型做启发式推荐，适合演示，但不足以支撑：

- 多轮调整
- 图表复用
- 钻取和联动
- 保存分析工件
- 输出稳定的分析表达结果

---

## 3. 目标能力

### 3.1 图表规划

- 根据问题意图和结果特征规划图表
- 输出结构化 chart spec

### 3.2 结果表达

- 一个请求返回统一的 analysis artifact
- 包含 query、summary、chart、export metadata

### 3.3 数据规模治理

- 大结果集采样
- 时间序列聚合
- top-k 压缩
- 明确的展示预算

### 3.4 交互支持

- 钻取
- 切换维度
- 切换图表类型
- 保存当前视图

---

## 4. 建议输出对象

建议新增统一结果对象，至少包含：

- `answer_summary`
- `result_table`
- `chart_plan`
- `chart_spec`
- `display_warnings`
- `export_payload`
- `source_sql`

---

## 5. 图表类型策略

当前 bar / line / pie / scatter / table 可以保留，但要扩成两层：

### 5.1 默认层

- 保留当前启发式图表推荐
- 用作低成本默认输出

### 5.2 规划层

- 由 LLM + 规则混合决定更适合的图表表达
- 输出 chart spec 和交互建议

---

## 6. 导出要求

导出不能只导出表格数据，还应支持导出：

- SQL / action 背景
- summary
- chart spec
- 参数与时间范围
- 结果快照

---

## 7. 实施顺序

### Step 1：统一 analysis artifact 对象

### Step 2：chart plan 与 chart spec 抽象

### Step 3：大结果集展示预算

### Step 4：与会话和导出打通

### Step 5：钻取和视图保存

---

## 8. 验收标准

- 图表规划结果可复现
- 结果对象可用于 UI 渲染和导出
- 大结果集不会直接拖垮展示层
- 多轮会话中可以继承上一轮图表上下文
