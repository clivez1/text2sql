# stage0-01. 文档治理与边界规则

> 状态：已完成
> 更新：2026-04-18

---

## 1. 根目录规则

1. 根目录只放 README 和 00 到 04 五份入口文档。
2. 根目录不放阶段执行清单、文件级迁移方案和主题长文。

---

## 2. 阶段目录规则

1. 每个阶段使用独立 stage 目录。
2. 每个阶段至少有一个 00-stage-plan.md 作为入口。
3. 阶段内的详细执行文档全部收纳在对应 stage 目录中。

---

## 3. 参考目录规则

1. 架构背景进入 references/architecture。
2. 能力主题和经验背景进入 references/themes。
3. references 只提供背景，不直接充当执行入口。

---

## 4. 进度文档规则

1. 已完成任务只保留在 01-completed.md。
2. 未完成任务只保留在 02-pending.md。
3. 当前阶段任务只保留在 03-in-progress.md。
4. 04-temp-memory.md 只存当前工作记忆，不存长期历史。
